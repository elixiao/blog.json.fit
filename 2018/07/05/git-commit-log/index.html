    Git 提交规范

    分为 header 和 body 两个部分：

    header

    header 由 type 和 subject 两部分组成：

    type

    表示类型，只能是下几种：

    • feat: 新功能
    • fix: 修复问题
    • docs: 修改文档
    • style: 修改代码格式，不影响代码逻辑
    • refactor: 重构代码，理论上不影响现有功能
    • perf: 提升性能
    • test: 增加修改测试用例
    • chore: 改变构建流程、或者增加依赖库、工具等
    • revert: 回滚到上一个版本

    subject

    这次提交的简要描述，不要超过一行。

    body

    这次提交的详细描述，是选填项，可以多行。
