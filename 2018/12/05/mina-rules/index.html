小程序开发准则

2018.12.05 李潇

使用 ES6 语法

  • 不要用 var，常量用 const，变量用 let，let 越少越好
  • 不要用 == 和 !=，请严格使用 === 和 !==

使用局部变量缓存反复查找的对象

如果反复查找某对象，且该对象是：

  • 全局变量
  • dom 查询结果
  • 作用域链较深的对象

那么一定要换成局部变量，千万不要出现 data.data.data.xxx，因为作用域链太深且反复使用。

函数要封装

坚决避免超长函数：

一个函数只干一件事，尽量不要超过20行。

变量命名要友好

变量命名：首字母小写，驼峰式命名，匈牙利命名
函数命名：首字母小写，驼峰式命名，统一使用动词或者动词+名词形式

匈牙利命名法语法：变量名＝类型＋对象描述

  • 类型：指变量的类型
  • 对象描述：指对象名字全称或名字的一部分，要求有明确含义，命名要容易记忆容易理解。

不要出现 data.data，res.data 就强很多。

本地存储

每个微信小程序都可以有自己的本地缓存，可以通过下面的函数对本地缓存进行设置、获取和清理。

  • wx.setStorage、wx.setStorageSync
  • wx.getStorage、wx.getStorageSync
  • wx.clearStorage、wx.clearStorageSync

同一个微信用户，同一个小程序 storage 上限为 10MB。localStorage 以用户维度隔离，同一台设备上，A 用户无法读取到 B 用户的数据。其中 localStorage 是持久存储的，但不建议将关键信息全部存在 localStorage，以防用户换设备的情况。

wxs 过滤器

在 pages/filter.wxs 中定义一些常用的过滤器，使用的时候在 wxml 中引用：

  1. formatDate 格式化日期

  1. formatMoney 格式化价格

其他过滤器也可以写在 filter.wxs 中，但是注意

  • 只能用 var 定义变量
  • new Date() 要换成 getDate()
  • new RegExp() 要换成 getRegExp()
  • 不能引用 js 文件，也不能被 js 文件引用
